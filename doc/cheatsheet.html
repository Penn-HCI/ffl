Selectors

Literal Selectors - $ LaTeX\LaTeXLATE​X $

The LaTeX\LaTeXLATE​X string should generally match what your write in LaTeX, while you can use wildcards (see below) to enable more flexible selection.

You can use wildcards in your literal selectors. To match any single token/group, use ?. For a sequence of any length, use *. Note that wildcards are bounded by {...} groups and * selects the longest possible match.

To match the literal character of the wildcards use \? for "?" or \* for "*".

Special Classes

You can also use special pre-configured classes in the same syntax as CSS (. prefix). The available list is as follows,

selector Description
.numerator any numerator of \frac
.denominator any denominator of \frac
.superscript any superscript after ^
.subscript any denominator after _
.constant any string consisting of only 0-9

Selector Combinators

You can combine selectors with some simple logical operations.Note that and has higher precedence than or, meaning that the selector section you can use for a block looks like either

  • a single selector as described above
  • a comma separated list of either single selectors or intersections
and Combinator - intersection(...)

intersection( selector,...,selector)

For example, intersection($x_\?$, .constant) would only select a constant subscript under xxx (e.g. 111 in x1x_1x1​ but not nnn in xnx_nxn​).

or Combinator - ,

selector,selector

For example, $x$, $y$ would only select any xxx and any yyy. It is the same as writing the same style block for both selectors.

Attributes

CSS Attributes

We expect most basic CSS properties as attribute-keys and values to work. Common ones you might need include color, background-color, etc. and their values follow the format of their CSS counter parts.

FFL Specific Attributes

attribute-key value
label any string terminated by the attribute-dividing ;
label-position above/below/auto
label-marker extend/line/none
